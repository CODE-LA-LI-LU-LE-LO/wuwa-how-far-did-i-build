# 목표 에코셋 검색형 선택 입력 기능 설명서

## 1. 배경과 가능 여부

현재 목표 에코셋은 정해진 드롭다운 목록에서 항목을 직접 고르는 방식이다. 제안한 방식처럼 에코셋 이름 영역을 `input` 기반 검색형 선택 컨트롤로 바꾸고, 사용자가 입력한 문구를 포함하는 에코셋만 아래 후보 목록에 표시한 뒤, 후보를 선택하면 왼쪽에는 에코셋 아이콘을 보여주고 오른쪽 입력칸에는 에코셋 전체 이름을 채우는 방식으로 구현할 수 있다.

이 기능은 기존 `echoSets` 목록의 `name`과 `icon` 데이터를 그대로 활용하면 되므로 별도 데이터 구조 변경 없이 UI와 이벤트 처리만 확장해 만들 수 있다.

## 2. 목표

- 목표 에코셋 이름 입력 경험을 긴 드롭다운 탐색에서 검색 후 선택 방식으로 개선한다.
- 사용자가 일부 단어만 입력해도 해당 문구가 들어간 에코셋 후보를 빠르게 찾을 수 있게 한다.
- 후보 선택 시 아이콘과 전체 에코셋 이름을 함께 확정해 기존 목표 저장 구조와 호환되게 한다.
- 마우스, 키보드, 모바일 터치 환경에서 모두 사용할 수 있는 접근성 있는 콤보박스로 제공한다.

## 3. 적용 범위

### 3.1 포함 범위

- 파밍 카드의 목표 에코셋 입력 행에 있는 에코셋 이름 컨트롤.
- 관리자 목표 모드와 수동 입력 목표 모드에서 편집 권한이 있는 경우의 입력 UI.
- 읽기 전용 상태에서 선택된 에코셋 아이콘과 이름을 표시하는 UI.
- 에코셋 후보 목록 필터링, 선택, 초기화, 저장 처리.

### 3.2 제외 범위

- 에코셋 데이터 자체의 추가, 삭제, 이름 변경.
- 목표 메인에코 입력 방식 변경.
- 현재 스테이터스의 에코셋 입력 방식 변경.
- 신규 자유 입력 에코셋 저장. 에코셋은 기존 목록 중 하나를 선택해야 한다.

## 4. 사용자 시나리오

1. 사용자가 목표 에코셋 이름 입력칸을 클릭한다.
2. 입력칸 아래에 에코셋 후보 목록이 열린다.
3. 사용자가 `구름`을 입력하면 이름에 `구름`이 포함된 에코셋만 후보로 표시된다.
4. 사용자가 후보 중 `떠오르는 구름`을 선택한다.
5. 입력칸 왼쪽에는 `떠오르는 구름` 아이콘이 표시되고, 입력값에는 `떠오르는 구름` 전체 이름이 입력된다.
6. 기존 목표 에코셋 저장 로직이 선택된 전체 이름을 저장한다.
7. 사용자가 다시 입력값을 수정하면 선택 확정 상태는 해제되고 후보 목록이 입력값 기준으로 다시 필터링된다.

## 5. 기능 요구사항

### 5.1 검색 입력

- 기존 에코셋 이름 드롭다운 버튼 위치를 검색 가능한 `input`으로 대체한다.
- 입력칸의 placeholder는 `에코셋 검색`으로 표시한다.
- 입력값은 대소문자와 앞뒤 공백을 무시해 검색한다.
- 한글 이름 검색은 부분 문자열 포함 방식으로 처리한다.
- 입력 중에는 아직 저장하지 않고, 후보를 선택했을 때만 목표 에코셋 이름을 저장한다.
- 입력값이 비어 있으면 전체 후보와 `선택 안 함` 항목을 표시한다.

### 5.2 후보 목록

- 후보 목록은 입력칸 바로 아래에 표시한다.
- 각 후보는 왼쪽 아이콘, 오른쪽 전체 에코셋 이름으로 구성한다.
- 후보는 `echoSets` 배열 순서를 유지한다.
- 검색어가 포함된 후보만 표시한다.
- 일치 후보가 없으면 `일치하는 에코셋이 없습니다` 안내를 표시하고 선택 가능한 에코셋 후보는 표시하지 않는다.
- `선택 안 함` 항목을 선택하면 저장값을 빈 문자열로 바꾸고 아이콘을 빈 상태로 표시한다.

### 5.3 선택 확정

- 후보를 클릭하거나 키보드로 선택하면 해당 후보의 전체 이름을 저장한다.
- 선택 확정 후 입력칸 값은 후보의 전체 이름으로 바뀐다.
- 선택 확정 후 왼쪽 아이콘은 후보의 아이콘으로 바뀐다.
- 선택 확정 후 후보 목록은 닫힌다.
- 기존 `goal.echoSets[index].name`에는 전체 에코셋 이름만 저장한다.

### 5.4 편집 권한과 읽기 전용 상태

- 목표 편집 권한이 없으면 기존처럼 아이콘과 이름만 표시하고 입력은 비활성화한다.
- 관리자 목표 편집 가능 상태와 수동 목표 편집 가능 상태에서만 검색 입력을 활성화한다.
- 비활성화 상태에서는 후보 목록이 열리지 않는다.

### 5.5 키보드 조작

- `Enter`: 현재 강조된 후보를 선택한다.
- `ArrowDown`: 다음 후보로 이동한다.
- `ArrowUp`: 이전 후보로 이동한다.
- `Escape`: 후보 목록을 닫고 기존 확정값으로 입력칸을 되돌린다.
- `Tab`: 후보 목록을 닫되 입력값이 확정된 후보가 아니면 기존 저장값을 유지한다.

### 5.6 포커스와 모바일 동작

- 입력칸 focus 시 기존 확정값을 임시로 보관한 뒤 입력칸을 비우고 전체 후보 목록을 연다.
- 입력칸 바깥을 클릭하면 후보 목록을 닫고, 새 후보를 선택하지 않았다면 임시로 보관한 기존 확정값을 복원한다.
- 모바일에서는 후보 목록의 최대 높이를 제한하고 내부 스크롤을 허용한다.
- 후보 항목은 터치하기 쉽도록 최소 높이 38px 이상을 유지한다.

## 6. 데이터 및 저장 정책

- 기존 저장 필드인 `goal.echoSets[].name`을 그대로 사용한다.
- 저장 가능한 값은 `echoSets`에 존재하는 에코셋 이름 또는 빈 문자열이다.
- 사용자가 입력만 하고 후보를 선택하지 않은 문자열은 저장하지 않는다.
- 기존 저장 데이터의 정규화 함수는 현재처럼 알 수 없는 에코셋 이름을 빈 문자열로 정규화한다.
- `goal.echoSet` 같은 과거 호환 필드는 기존 정규화 흐름을 유지한다.

## 7. UI 구성안

```text
[아이콘] [에코셋 검색 input                  v]
        ┌────────────────────────────────────┐
        │ [아이콘] 떠오르는 구름              │
        │ [아이콘] 구름을 걷는 달빛           │
        └────────────────────────────────────┘
```

- 선택 전 또는 선택 안 함 상태: 왼쪽에 빈 원형 아이콘 자리 표시자를 보여준다.
- 선택 후: 왼쪽에 선택한 에코셋 아이콘을 보여준다.
- 입력칸에는 선택된 전체 이름을 표시한다.
- 후보 목록은 기존 드롭다운 스타일을 최대한 재사용한다.

## 8. 구현 메모

- 기존 `renderEchoSetPicker(characterId, index, value, enabled)`를 검색형 콤보박스 렌더러로 변경한다.
- 각 컨트롤에는 캐릭터 ID와 에코셋 인덱스를 알 수 있는 data attribute를 유지한다.
- 입력 이벤트는 화면 필터링만 처리하고 저장 함수는 호출하지 않는다.
- 후보 선택 이벤트는 기존 `updateGoalEchoSet(id, index, "name", value)`를 호출한다.
- 렌더 후 이벤트 바인딩 위치는 기존 `[data-echo-set-option]` 처리 흐름을 확장한다.
- 열림 상태, 검색어, 강조 후보 인덱스는 저장 데이터가 아니라 DOM 상태로 관리한다.
- 선택 후 `renderRoster()`가 호출되면 확정된 값 기준으로 새 UI가 다시 렌더링된다.

## 9. 예외 처리

- 검색어에 매칭되는 후보가 없는 경우 저장값은 변경하지 않는다.
- 사용자가 입력한 뒤 blur했지만 후보를 선택하지 않은 경우, 입력칸은 기존 저장값으로 복원한다.
- 저장된 에코셋 이름에 해당하는 아이콘을 찾지 못하면 `선택 안 함`과 동일한 빈 아이콘을 표시한다.
- 에코셋 목록이 비어 있는 경우 `선택 가능한 에코셋이 없습니다` 안내를 표시한다.

## 10. 완료 기준

- 에코셋 이름 입력칸에 검색어를 입력하면 후보 목록이 부분 일치로 필터링된다.
- 후보 선택 시 아이콘과 전체 이름이 입력칸에 반영된다.
- 선택한 값이 기존 목표 데이터 구조에 저장된다.
- 편집 불가 상태에서는 입력과 후보 선택이 불가능하다.
- 마우스, 터치, 키보드로 후보를 선택할 수 있다.
- 기존 목표 비교, 목표 JSON 다운로드, 개인 상태 저장 동작이 깨지지 않는다.

## 11. 테스트 체크리스트

- `구름` 입력 시 `구름`이 포함된 후보만 보이는지 확인한다.
- 후보 선택 후 입력칸에 전체 이름과 아이콘이 표시되는지 확인한다.
- `선택 안 함` 선택 시 저장값이 빈 문자열로 바뀌는지 확인한다.
- 후보를 선택하지 않고 blur하면 기존 저장값이 유지되는지 확인한다.
- 관리자 목표 편집 모드와 수동 목표 편집 모드에서 모두 동작하는지 확인한다.
- 편집 권한이 없는 상태에서는 검색 입력이 비활성화되는지 확인한다.
- 키보드 `ArrowUp`, `ArrowDown`, `Enter`, `Escape`, `Tab` 동작을 확인한다.
- 모바일 폭에서 후보 목록이 화면 밖으로 과도하게 넘치지 않는지 확인한다.

## 12. 작업 순서

아래 순서는 바로 구현에 들어갈 수 있도록 선행 관계가 있는 작업부터 작게 나눈 실행 단위이다. 각 단계는 완료 확인 기준을 통과한 뒤 다음 단계로 넘어간다.

### 12.1 현재 에코셋 선택 UI의 렌더링 위치 확인

- 작업 내용
  - `app.js`에서 목표 에코셋 행을 그리는 `renderGoalEchoSetRow()`와 에코셋 선택 컨트롤을 그리는 `renderEchoSetPicker()`의 호출 관계를 확인한다.
  - 현재 후보 선택 버튼이 사용하는 `data-echo-set-option`, `data-character`, `data-index`, `data-value` 속성을 확인한다.
  - 현재 저장 함수 `updateGoalEchoSet(id, index, "name", value)`가 후보 선택 후 호출되는 흐름을 확인한다.
- 완료 확인 기준
  - 변경해야 할 렌더 함수, 이벤트 바인딩 위치, 저장 함수가 각각 하나씩 식별되어 있다.
  - 데이터 저장 필드를 새로 만들 필요가 없다는 점을 확인했다.

### 12.2 콤보박스 DOM 구조를 먼저 렌더링한다

- 작업 내용
  - `renderEchoSetPicker()`의 편집 가능 상태 UI를 `details/summary` 기반 드롭다운에서 아이콘 + `input` + 후보 목록 컨테이너 구조로 바꾼다.
  - 기존 읽기 전용 UI는 유지한다.
  - 선택된 값이 있으면 input value에 전체 이름을 넣고, 선택된 아이콘을 왼쪽에 표시한다.
  - 선택값이 없으면 빈 아이콘 자리와 빈 input 또는 placeholder를 표시한다.
- 완료 확인 기준
  - 편집 가능 상태에서 에코셋 이름 영역이 input처럼 보인다.
  - 기존 선택값이 있는 캐릭터는 렌더 직후 input에 전체 에코셋 이름이 보인다.
  - 편집 불가 상태의 표시 방식은 기존과 동일하게 유지된다.

### 12.3 후보 목록을 전체 목록으로 표시한다

- 작업 내용
  - input 아래 후보 목록에 `선택 안 함`과 모든 `echoSets` 항목을 렌더링한다.
  - 각 후보는 기존처럼 `data-echo-set-option`, `data-character`, `data-index`, `data-value`를 가진 버튼으로 만든다.
  - 후보 항목에는 아이콘과 전체 이름을 함께 표시한다.
- 완료 확인 기준
  - input focus 또는 클릭 시 후보 목록이 열린다.
  - 후보 목록에 `선택 안 함`과 모든 에코셋이 아이콘과 함께 보인다.
  - 후보 클릭 시 기존 `updateGoalEchoSet()` 흐름으로 값이 저장된다.

### 12.4 입력값으로 후보를 필터링한다

- 작업 내용
  - 에코셋 검색 input에 `input` 이벤트를 연결한다.
  - 검색어는 `trim().toLowerCase()` 기준으로 정규화한다.
  - 각 후보의 에코셋 이름도 같은 기준으로 비교해 검색어가 포함된 항목만 표시한다.
  - 입력값이 비어 있으면 전체 후보를 다시 표시한다.
- 완료 확인 기준
  - `구름`을 입력하면 이름에 `구름`이 들어간 후보만 남는다.
  - 검색어를 지우면 전체 후보가 다시 보인다.
  - 입력 중에는 저장 데이터가 바뀌지 않는다.

### 12.5 후보 없음 상태를 처리한다

- 작업 내용
  - 검색 결과가 0개이면 후보 목록에 `일치하는 에코셋이 없습니다` 안내를 표시한다.
  - 이 안내는 버튼이 아니어야 하며 선택 이벤트가 발생하지 않아야 한다.
  - `선택 안 함`은 입력값이 비어 있을 때만 상단에 표시한다.
- 완료 확인 기준
  - 존재하지 않는 검색어를 입력하면 안내 문구만 표시된다.
  - 안내 문구를 클릭해도 저장값은 변경되지 않는다.
  - 검색어를 지우면 다시 전체 후보와 `선택 안 함`이 보인다.

### 12.6 선택 확정 후 표시와 저장 흐름을 정리한다

- 작업 내용
  - 후보 선택 시 input value를 후보의 전체 이름으로 바꾼다.
  - 후보 선택 시 왼쪽 아이콘을 선택한 후보 아이콘으로 바꾼다.
  - 후보 선택 후 목록을 닫는다.
  - 기존처럼 `updateGoalEchoSet(id, index, "name", value)`를 호출해 저장과 재렌더를 수행한다.
- 완료 확인 기준
  - 후보를 클릭하면 input에 후보 전체 이름이 들어간다.
  - 후보를 클릭하면 왼쪽 아이콘이 선택한 에코셋 아이콘으로 바뀐다.
  - 새로고침 또는 재렌더 후에도 선택한 에코셋이 유지된다.

### 12.7 미확정 입력값 복원 규칙을 추가한다

- 작업 내용
  - focus 시 기존 저장값을 임시 보관하고 input을 비워 전체 후보를 보여준다.
  - 사용자가 검색어를 입력했지만 후보를 선택하지 않고 blur하면 기존 저장값으로 input을 되돌린다.
  - `Escape`를 누른 경우도 기존 저장값으로 되돌리고 목록을 닫는다.
  - `Tab`으로 이동할 때는 저장값을 변경하지 않고 목록만 닫는다.
- 완료 확인 기준
  - 저장된 값이 `떠오르는 구름`일 때 다시 focus하면 input은 일시적으로 비워지고 전체 후보가 표시된다.
  - 저장된 값이 `떠오르는 구름`일 때 `구`까지만 입력하고 바깥을 클릭하면 input이 `떠오르는 구름`으로 복원된다.
  - 후보를 선택하지 않은 문자열은 `goal.echoSets[].name`에 저장되지 않는다.
  - `Escape`와 `Tab` 동작이 저장값을 임의 변경하지 않는다.

### 12.8 키보드 후보 이동과 선택을 구현한다

- 작업 내용
  - `ArrowDown`으로 다음 후보를 강조한다.
  - `ArrowUp`으로 이전 후보를 강조한다.
  - `Enter`로 현재 강조된 후보를 선택한다.
  - 후보 강조 상태는 저장 데이터가 아니라 DOM 상태로만 관리한다.
- 완료 확인 기준
  - 키보드만으로 후보를 이동하고 선택할 수 있다.
  - `Enter`로 선택한 값도 마우스 클릭과 동일하게 저장된다.
  - 검색어 변경 시 강조 후보가 첫 번째 후보로 초기화된다.

### 12.9 스타일을 정리한다

- 작업 내용
  - 기존 `.echo-picker`, `.echo-picker-button`, `.echo-picker-options` 스타일을 최대한 재사용한다.
  - input이 포함된 새 컨트롤의 그리드 구조를 아이콘, 입력칸, 열림 표시 영역으로 맞춘다.
  - 후보 목록의 최대 높이와 스크롤을 유지한다.
  - 모바일 폭에서 후보 목록이 카드 영역 밖으로 과하게 넘치지 않도록 확인한다.
- 완료 확인 기준
  - 기존 드롭다운과 시각적으로 크게 이질적이지 않다.
  - 후보 버튼의 아이콘과 텍스트 정렬이 깨지지 않는다.
  - 모바일 폭에서도 후보 목록을 스크롤할 수 있다.

### 12.10 기존 목표 편집 권한 조건을 재검증한다

- 작업 내용
  - 관리자 목표 편집 가능 상태에서 검색형 입력이 활성화되는지 확인한다.
  - 수동 목표 편집 가능 상태에서 검색형 입력이 활성화되는지 확인한다.
  - 편집 권한이 없는 상태에서는 input과 후보 목록이 비활성화되는지 확인한다.
- 완료 확인 기준
  - 관리자 목표 편집 버튼을 켠 카드에서만 관리자 목표 에코셋을 변경할 수 있다.
  - 수동 입력 목표 편집 버튼을 켠 카드에서만 사용자 목표 에코셋을 변경할 수 있다.
  - 편집 권한이 없는 카드에서는 후보 목록이 열리지 않는다.

### 12.11 저장, 다운로드, 복원 흐름을 확인한다

- 작업 내용
  - 후보 선택 후 로컬 저장 상태에 선택한 전체 이름이 들어가는지 확인한다.
  - 관리자 목표 JSON 다운로드에 선택한 이름이 들어가는지 확인한다.
  - 기존 저장 데이터를 로드했을 때 아이콘과 input 값이 정상 표시되는지 확인한다.
- 완료 확인 기준
  - `goal.echoSets[].name` 값이 선택한 전체 에코셋 이름으로 저장된다.
  - 목표 JSON 다운로드 결과에 선택한 이름이 유지된다.
  - 앱을 새로 열어도 선택한 에코셋이 표시된다.

### 12.12 최종 검증을 실행한다

- 작업 내용
  - 정적 검증 스크립트를 실행한다.
  - 대표 수동 시나리오를 한 번씩 확인한다.
  - UI 변경이 있으므로 가능하면 데스크톱 폭과 모바일 폭 스크린샷을 남긴다.
- 완료 확인 기준
  - `node scripts/verify.mjs`가 통과한다.
  - `구름` 검색, 후보 선택, 선택 안 함, 후보 없음, blur 복원, 키보드 선택이 모두 기대대로 동작한다.
  - 변경된 UI 스크린샷에서 input, 아이콘, 후보 목록이 정상적으로 보인다.
