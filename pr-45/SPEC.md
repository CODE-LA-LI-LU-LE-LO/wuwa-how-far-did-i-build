# 내가 어디까지 잘 키웠더라 기능 설명서

## 1. 제품 목표

명조 캐릭터의 보유 여부를 체크로 관리하고, 보유 여부와 관계없이 파밍 상태와 목표 달성 여부를 기록하는 반응형 웹사이트를 만든다.

주요 사용자는 사이트 제작자 본인, 공유 링크를 받은 사람, Google 계정으로 로그인한 사용자다. 화면은 깔끔하고 보기 쉬워야 하며, 캐릭터 카드와 파밍 현황은 시각적으로 미려하게 표현한다.

## 2. 사용자 구분

### 일반 사용자

- 캐릭터 보유 여부를 변경한다.
- 보유 캐릭터의 파밍 상태를 입력한다.
- 목표 스테이터스와 현재 스테이터스를 비교한다.
- 목표 달성 여부를 직접 체크할 수 있다.
- 캐릭터 목록을 정렬하고 필터링한다.
- 기록을 로컬 저장, 공유 링크, Google 로그인 저장으로 관리한다.

### 관리자

- 캐릭터 기본 데이터를 추가, 편집, 제거한다.
- 캐릭터별 목표 스테이터스 기본값을 입력한다.
- 일반 사용자가 참고할 기본 파밍 목표를 관리한다.

관리자 기능은 초기 버전에서는 별도 관리자 화면 또는 설정 모드로 제공할 수 있다. 관리자 인증 방식은 추후 결정한다.

## 3. 꼭 필요한 기능

### 3.1 캐릭터 보유 유무 체크 관리 탭

캐릭터 전체 목록을 보고, 각 캐릭터의 보유 여부를 빠르게 변경하는 탭이다.

#### 관리자용 기능

관리자는 캐릭터 기본 정보를 관리할 수 있다.

입력 항목:

- 캐릭터명
- 캐릭터 이미지 파일
- 무기
- 속성
- 공개 여부

무기 선택값:

- 직검
- 대검
- 권총
- 권갑
- 증폭기

속성 선택값:

- 기류
- 용융
- 응결
- 전도
- 인멸
- 회절

공개 여부 선택값:

- 공개
- 비공개

관리 동작:

- 공개 여부 설정
- 캐릭터 추가
- 캐릭터 편집
- 캐릭터 제거
- 이미지 파일 교체
- 필수값 누락 시 저장 불가

#### 사용자용 기능

사용자는 캐릭터 목록에서 보유 여부를 변경할 수 있다.

보유 상태 변경:

- 캐릭터 카드에 자물쇠 버튼을 표시한다.
- 잠김 상태는 미보유를 의미한다.
- 잠금 해제 상태는 보유를 의미한다.
- 보유 여부는 필터와 상태 표시 기준으로 사용하며, 상세 기록 가능 여부를 제한하지 않는다.

정렬 기준:

- 이름
- 속성
- 무기

정렬 UI:

- 화면 상단에 정렬 기준 버튼을 표시한다.
- 정렬 기준 버튼을 클릭하면 캐릭터 목록의 정렬 방식이 변경된다.
- 속성 정렬 선택 시 도구막대 바로 하단의 가로 카테고리 영역에 속성 그룹이 표시된다.
- 무기 또는 등급 정렬 선택 시 동일한 가로 카테고리 영역이 해당 그룹으로 변경된다.
- 이름 정렬 선택 시 카테고리 영역 없이 캐릭터가 이름 순으로 나열된다.
- 모바일 레이아웃에서는 카테고리 영역의 텍스트 라벨을 숨기고 아이콘 또는 기호만 표시한다.

정렬 동작:

- 이름 정렬은 캐릭터명을 기준으로 오름차순 정렬한다.
- 속성 정렬은 속성 그룹별로 묶고, 그룹 내부는 이름 순으로 정렬한다.

### 3.2 캐릭터 파밍 상태 관리 탭

보유 여부와 관계없이 캐릭터의 현재 파밍 상태를 입력하고, 목표 스테이터스와 비교하는 탭이다.

#### 관리자용 기능

관리자는 캐릭터별 목표 스테이터스 기본값을 입력한다.

목표 스테이터스 입력 항목:

- 에코셋
  - 목표 에코셋 이름은 검색형 선택 입력으로 제공한다. 사용자가 입력한 문구를 포함하는 에코셋 후보를 아래 목록에 표시하고, 후보 선택 시 아이콘과 전체 에코셋 이름을 입력칸에 반영한다. 상세 기능 설명은 `docs/echo-set-combobox-feature-spec.md`를 따른다.
- 메인에코
- 주옵
  - 에코 구성 및 목표 수치의 분기 선택값은 `-`, `A`, `B`, `C`, `D`, `E`, `F`를 제공한다.
  - 에코 구성과 수치입력 표의 주옵 선택 열은 화면에서 `속성`으로 표시하고, 선택 드롭박스는 마우스/터치뿐 아니라 `ArrowUp`, `ArrowDown`, `Home`, `End`, `PageUp`, `PageDown`, `Enter`, `Escape`, `Tab` 키보드 조작을 지원한다.
- 권장수치
  - 수치 입력란은 키보드 상하 방향키 조작 시 5 단위로 증감한다.
- 비고

목표 데이터는 캐릭터별로 저장된다. 일반 사용자는 이 값을 기본 목표로 불러와 자신의 현재 스테이터스와 비교할 수 있다.

#### 사용자용 기능

사용자는 보유 여부와 관계없이 캐릭터의 현재 스테이터스를 입력한다.

현재 스테이터스 입력 항목:

- 에코셋
  - 현재 스테이터스의 에코셋 입력 방식은 이번 검색형 목표 에코셋 기능 범위에 포함하지 않는다.
- 메인에코
- 주옵
- 현재 수치
  - 수치 입력란은 키보드 상하 방향키 조작 시 5 단위로 증감한다.
  - 수치입력 표의 내 캐릭터 열에서 분기만 다르고 속성이 같은 현재 수치 항목이 여러 개 존재하는 경우, 한 항목의 입력값을 포커스 해제 시 동일 속성의 다른 항목에도 반영한다.
  - 동일 속성 항목 간 입력값이 다를 때는 마지막으로 포커스 해제된 입력값이 우선하며, 기존 입력값을 덮어쓴다.
- 비고 또는 메모

목표 비교:

- 현재 스테이터스와 목표 스테이터스를 같은 항목끼리 비교한다.
- 목표 조건을 만족하면 해당 항목을 목표 달성으로 표시한다.
- 목표 조건을 만족하지 못하면 미달성으로 표시한다.
- 사용자가 직접 목표 달성 처리 체크를 켤 수 있다.
- 사용자 직접 체크는 자동 판정보다 우선한다.

목표 전환:

- 기본 목표는 관리자가 입력한 목표 스테이터스를 사용한다.
- 사용자는 임의 목표 스테이터스를 직접 입력할 수 있다.
- 토글로 관리자 목표와 사용자 임의 목표를 전환한다.
- 사용자 임의 목표는 개인 저장 데이터에 포함한다.

캐릭터 보기 필터:

보유 관리 탭:

- 전체
- 보유
- 미보유

파밍 관리 탭:

- 전체
- 보유
- 미보유
- 목표달성
- 미달성

필터 동작:

- 보유 관리 탭에서는 전체/보유/미보유 중 하나를 선택한다.
- 파밍 관리 탭에서는 여러 필터를 동시에 체크할 수 있다.
- 파밍 관리 탭에서 보유와 미보유를 함께 선택하면 전체 보유 상태가 표시된다.
- 파밍 관리 탭에서 목표달성과 미달성을 함께 선택하면 전체 목표 상태가 표시된다.
- 보유/미보유 상태는 필터와 상태 표시 기준으로만 사용한다.
- 보유 여부와 관계없이 캐릭터 카드에서 상태 확인 및 상세 입력이 가능하다.

정렬 기준:

- 캐릭터 보유 유무 체크 관리 탭과 동일하게 이름, 속성, 무기, 등급 기준을 사용한다.
- 속성, 무기, 등급 정렬 시 도구막대 하단 가로 카테고리 영역도 동일하게 변경한다.
- 보유 관리 탭과 파밍 관리 탭을 전환해도 등급/속성/무기 상세 카테고리 필터 값은 유지한다.

### 3.3 파밍 상태 카드 표시

캐릭터 파밍 상태 관리 탭에는 캐릭터별 카드가 표시된다. 첨부 이미지의 표 형태를 웹 카드로 재구성한다.

카드 주요 영역:

- 캐릭터 이름
- 캐릭터 이미지
- 에코셋
- 메인에코
- 목표 주옵 목록
- 목표 권장수치
- 현재 입력값
- 목표 달성 여부
- 비고

카드 예시 데이터 구조:

- 캐릭터명: 루실라
- 에코셋: 소리 없이 내려앉은 기도의 눈 / 떠오르는 구름
- 메인에코: 글로모스 3코 / 구름 세트는 타종 or 백로
- 주옵: 크리피해, 응결피증, 공격력%
- 권장수치: 크리확률 75, 크리피해 265, 공명효율 0, 공격력 2000
- 사용자 현재값: 크리확률 80, 크리피해 255, 공명효율 0, 공격력 2000
- 비고: 딜 기여도 낮음

시각 표시:

- 목표를 만족한 항목은 강조 색상으로 표시한다.
- 목표에 미달한 항목은 경고 또는 중립 색상으로 표시한다.
- 사용자가 직접 완료 체크한 캐릭터는 목표달성 상태로 표시한다.
- 모바일에서는 표를 그대로 축소하지 않고, 항목별 카드 또는 섹션으로 접어 표시한다.

### 3.4 저장과 공유

저장 방식:

- 비로그인 사용자는 브라우저 로컬 저장소에 자동 저장한다.
- Google 로그인 사용자는 계정별 클라우드 저장을 사용한다.
- 공유 링크 사용자는 링크에 포함된 스냅샷을 볼 수 있다.

공유 링크:

- 공유 링크를 받은 사람은 해당 기록을 볼 수 있다.
- 공유 받은 기록은 바로 원본 저장소를 덮어쓰지 않는다.
- 사용자가 저장 버튼을 누르면 자신의 기록으로 저장한다.

Google 로그인:

- Google 계정으로 로그인할 수 있다.
- 로그인 사용자는 같은 계정으로 접속하면 저장된 기록을 불러온다.
- 계정별 데이터는 다른 사용자가 읽거나 쓸 수 없어야 한다.

## 4. 나중에 해도 되는 기능

### 다국어 지원

초기 버전은 한국어를 기준으로 만든다.

추후 지원 후보:

- 한국어
- 영어
- 일본어
- 중국어

다국어 적용 범위:

- UI 문구
- 필터와 정렬명
- 속성명
- 무기명
- 도움말

캐릭터명과 게임 고유명사는 별도 번역 테이블을 둘 수 있다.

## 5. 우선순위

1. 캐릭터 기본 데이터 구조 정리
2. 보유 유무 체크 관리 탭 구현
3. 파밍 상태 관리 탭 구현
4. 목표 스테이터스와 현재 스테이터스 비교
5. 필터, 정렬, 도구막대 하단 가로 카테고리 UI 구현
6. 공유 링크와 Google 로그인 저장 안정화
7. 관리자 편집 UI 정리
8. 다국어 지원

## 6. 완료 기준

- 캐릭터를 보유 상태로 변경할 수 있다.
- 보유 여부와 관계없이 파밍 상태를 입력할 수 있다.
- 관리자가 캐릭터 기본 정보와 목표 스테이터스를 관리할 수 있다.
- 사용자가 현재 스테이터스를 입력하고 목표와 비교할 수 있다.
- 사용자가 관리자 목표와 임의 목표를 전환할 수 있다.
- 보유 관리 탭은 전체/보유/미보유 단일 필터를 제공하고, 파밍 관리 탭은 전체/보유/미보유/목표달성/미달성 다중 필터를 제공한다.
- 이름, 등급, 속성, 무기 정렬이 두 탭에서 동일하게 동작한다.
- 우선순위 캐릭터 카드는 도구막대의 이름 검색, 카테고리, 보유/파밍/공개 필터와 정렬 기준을 동일하게 반영하되 목표달성 캐릭터는 항상 마지막에 표시한다.
- 앱 진입 시 우측 상세 패널은 보유 캐릭터 중 현재 캐릭터 정렬 언어 기준 이름 오름차순으로 가장 앞선 캐릭터를 최초 표시하며, 기본 정렬 언어는 한국어(`ko`)로 둔다.
- 공유 링크와 Google 로그인 저장 흐름이 동작한다.
- 모바일과 데스크톱에서 카드와 표 정보가 읽기 좋게 표시된다.
